# 智能医学分诊 RAG 系统 — 项目总结

## 项目背景

在医疗资源紧张、患者挂号难的现实背景下，患者往往不知道自己的症状该挂哪个科室，医院也缺乏智能化的预检分诊手段。本项目旨在构建一套 **基于大模型检索增强生成（RAG）的智能医学分诊系统**，通过整合医学教材知识库、知识图谱和医院科室数据，实现从症状输入→追问澄清→混合检索→LLM 推理→科室推荐的全流程自动化分诊，帮助患者快速找到合适的就诊科室，提高分诊效率和准确性。

## 技术栈

### 核心框架与语言

| 领域 | 技术 |
|------|------|
| 编程语言 | Python 3.10+ |
| Web 框架 | Flask（RESTful API + 前端页面） |
| 大模型接入 | LangChain（ChatOpenAI 统一接口） |

### AI / 大模型

| 模型/服务 | 用途 |
|-----------|------|
| **通义千问 qwen-turbo**（DashScope） | 默认 LLM，用于分诊推理、症状追问、Rerank 精排 |
| **DeepSeek-Chat** | 备选 LLM |
| **智谱 GLM-4** | 备选 LLM |
| **paraphrase-multilingual-MiniLM-L12-v2**（SentenceTransformer） | 多语言文本嵌入模型，384 维向量 |

### 向量检索与知识库

| 组件 | 用途 |
|------|------|
| **Faiss**（faiss-cpu） | 高效向量相似度搜索，IP（内积）索引，19,225 条医学知识向量 |
| **混合检索** | Faiss 语义检索（权重 0.7）+ 关键词倒排索引（权重 0.3） |
| **LLM Rerank 精排** | 对混合检索候选结果进行二次排序，提升 Top-3 准确率 |
| **NetworkX** | 构建医学知识图谱（疾病→症状→药物→科室关系网络），GraphML 格式存储 |

### 数据与资源

| 资源 | 规模 |
|------|------|
| 医学教材知识库 | 约 19,225 个文本块（chunks），来源包括《CT 诊断学》《临床药物治疗学》《外科学》等 |
| Faiss 向量索引 | 384 维，约 19,225 条向量 |
| 知识图谱 | 疾病-症状-药物-科室多类实体及关系网络 |
| 医院数据 | 虚构"阳光综合医院"（三级甲等），含 5 栋楼宇、全部临床/医技科室、医生排班及专长信息 |

## 主要工作内容

### 1. 医学知识库构建

- 解析多本医学教材（LaTeX 源文件），按标题层级（h1/h2/h3/h4）分割为语义完整的文本块
- 使用 SentenceTransformer 对文本块进行向量化，构建 Faiss 索引
- 构建中文字词倒排索引，支持关键词匹配
- 评估不同检索策略（稠密/稀疏/混合）的命中率、MRR、NDCG 等指标

### 2. 混合检索 + Rerank 系统

- 实现 **Faiss 向量检索（语义相似度）+ 关键词倒排检索** 的混合搜索策略
- 权重配置：语义检索 70%、关键词匹配 30%
- 两阶段筛选：先混合检索取 Top-60，再 LLM Rerank 精排取 Top-5
- 相比纯向量检索，混合检索 + Rerank 在相关性上显著提升

### 3. 医学知识图谱

- 基于 NetworkX 构建疾病-症状-药物-科室的实体关系图
- 支持实体关联查询：给定疾病，可获取常见症状、推荐药物、关联科室
- 以 GraphML 格式持久化存储

### 4. 多轮症状追问子系统

- 模拟医生问诊流程，通过多轮对话追问患者症状细节
- LLM 动态生成追问问题，自动避免重复提问
- 最多 5 轮追问，支持提前结束
- 追问结束后自动整理格式化症状摘要，作为分诊输入

### 5. 分诊推理与 Agent

- **ReAct 范式 Agent**：思考（Thought）- 行动（Action，搜索知识图谱/向量库/分析症状）- 观察（Observation）循环
- **优化版分诊 Agent**：基于混合检索上下文 + 知识图谱关联信息，LLM 综合分析
- 支持紧急程度分级：危急（红色）/ 紧急（橙色）/ 次紧急（黄色）/ 非紧急（绿色）
- 结合医院科室和医生信息进行推荐

### 6. 医院与智能分流

- 加载医院科室数据（科室位置、亚专科、医生职称/专长/排班）
- 根据症状匹配推荐科室和医生
- 智能分流：结合当前时间、医生排班和症状匹配度推荐最合适的医生

### 7. Flask Web 服务

- RESTful API 设计，涵盖分诊、症状追问、医院信息、科室推荐等接口
- 单页面前端，三步交互流程：描述症状 → 追问回答 → 查看分诊结果
- 支持 JSON 格式数据交换，跨域访问（CORS）

### 8. 性能优化

- 消除重复的 Rerank LLM 调用，一次分诊从 3 次 LLM 调用优化为 1 次
- 根据场景（上下文检索 vs 最终展示）选择是否启用 Rerank
- LLM max_tokens 从 2000 优化为 1000，缩短生成时间
